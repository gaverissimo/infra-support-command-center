01 Fundamentals & ITIL Workflow

Support Workflow & Processes
  • Troubleshooting Mindset Aja com base em evidências (logs). Isole o problema central antes de aplicar soluções "bala de prata".
  • OSI Reference Model Quick Check L1 (Físico) -> L2 (MAC) -> L3 (IP) -> L4 (Porta TCP/UDP) -> L7 (App HTTP).
  • Incident Mgmt (Break-fix) Foco total em restaurar o serviço o mais rápido possível ao usuário (Resolução de contorno permitida).
  • Problem Mgmt (Root Cause) Ação investigativa: Por que o incidente aconteceu? Achar a causa para evitar que volte de forma definitiva.
  • Change Mgmt (RFC) Controle de ambiente: Planejamento, Rollback e Impacto visualizados e aprovados antes de alterar a infra.
Priorização & SLAs
  • Priority Matrix Cruzamento cego de Urgência (Tempo necessário para o negócio parar) x Impacto (Danos do negócio / Nº de Usuários).
  • SLA Awareness Acordo de Nível de Serviço. Entenda e monitore tempos tolerados de Resposta e Solução evitando estouro de contratos corporativos.
Documentation Pattern
  • Symptom Registrar o que exatamente falhou ou travou na percepção do usuário requisitante.
  • Scope Dimensione o dano. Afeta só a máquina da sala X ou o link inteiro corporativo?
  • Root cause Investigação real provada em logs de por que a falha original foi iniciada.
  • Fix Ação exata executada em terminal ou painel para contornar ou eliminar totalmente o problema.
  • Prevention O que faremos agora: Adicionar alerta? Redundância? Rotina de backup modificada?

Tools that help in real troubleshooting

RCA (Root Cause Analysis) Frequente em "Problems". Usa métodos como os 5-Whys e diagrama de Ishikawa.
Knowledge Base & ITSM ServiceNow, Jira Service Management, GLPI. Garantem visibilidade global dos tickets e SLAs.

02 Networking

The Connectivity Ladder
  • ping 127.0.0.1 Testa se a própria placa do PC responde (Pilha TCP/IP OK?).
  • ping local IP Testa se conectou na porta LAN local.
  • ping gateway Testa se o PC alcança o roteador (porta da rede pra fora). Se o gateway falhar, isolamento local.
  • ping DNS Testa se é possível contatar a máquina listada nos resolvadores de nome DHCP (interno ou provedor).
  • ping internet Testa a conexão saltando o gateway, usando IP limpo sem DNS (ex: ping 8.8.8.8).
  • ping destination Testa alcançar a aplicação ou máquina final (ex: ping srv-erp-01).
DNS vs Network Detection Trick
  • ping 8.8.8.8 works MAS ping google.com fails = DNS Issue! Se ping direto pra IP responde limpo, o link de internet e a placa de rede (Routing) estão 100% íntegros. O servidor DNS não consegue traduzir os nomes.
Port Testing Checklist
  • Test-NetConnection host -Port 443 (Windows Powershell) Retorna detalhadamente "TcpTestSucceeded : True" se a porta abrir ping localmente.
  • nc -zv host 443 (Linux/Mac) Zero-I/O Netcat testa silenciosamente se a porta remota TCP atende sockets no host.
  • telnet host 443 Diagnóstico de Socket clássico. Se ficar "Connection refused", há bloqueio de FireWall ou daemon caído.
Estrutura Local (Gateway, ARP e Rotas)
  • ipconfig /all | ip a Identifica local IP, Gateway Padrão e DNS Server local.
  • arp -a MAC cache: Tabela descobrindo Quem é quem na LAN. Essencial pra diagnosticar duplicação de IPs locais.
  • route print | ip route Mapeia Tabela de Roteamento interna (garante que X gateway empurra para X rota corporativa interna vs Internet).
  • tracert / traceroute Mapeia hops até o destino final informando a latência individual por Hop. Causa de loop de roteador.

Tools that help in real troubleshooting

Wireshark Analisa pacotes da rede interceptando frames (DNS, TCP, TLS, HTTP).
latência • pacotes perdidos • falha DNS • handshake TLS • timeouts
PingPlotter Visualiza a perda de pacotes e latência entre "hops" através de gráficos mantidos.
VPN lenta • link instável • jitter • problemas ISP
Advanced IP Scanner Inventaria e descobre ativos que estão mortos mas responderão porta ICMP limitados subnets locais.

03 DNS Troubleshooting

Local Cache Troubleshooting
  • ipconfig /flushdns Limpa cache DNS local (resolve stale records).
  • ipconfig /displaydns Mostra o cache DNS local atualmente armazenado na máquina.
  • ipconfig /registerdns Força o registro e renovação do IP no DNS AD.
Query Troubleshooting
  • nslookup google.com Consulta usando o DNS padrão configurado na máquina alvo.
  • nslookup google.com 8.8.8.8 Força a consulta utilizando um DNS específico/diferente.
  • dig +trace google.com (Linux/macOS/WSL) Consulta DNS excelente demonstrando o caminho e autoridades da zona.
DNS Resolution Flow & Logic
  • Resolution Flow 1. Hosts file → 2. DNS cache → 3. Local DNS server → 4. External DNS server.
  • Quick DNS Detection Logic ping 8.8.8.8 works MAS ping google.com fails = Garantia de problema no DNS!
Active Directory DNS Issues
  • User cannot login domain Use ipconfig /all. Esperado: O DNS é o IP do Domain Controller. NUNCA: DNS apontando pra 8.8.8.8 publicamente (em ambiente AD a autenticação quebra).
Record Type Troubleshooting
  • A hostname → IPv4
  • AAAA hostname → IPv6
  • MX mail routing (Responsável pelo destino de e-mails corporativos).
  • CNAME alias (Apelido apontando para um record A).
  • NS name server (Servidor que tem a autoridade sobre uma zona).
  • TXT Múltiplos usos (Geralmente SPF / regras de envio de e-mails blindado / Domain Verifications).
  • SRV domain services (Essencial para Login de Active Directory achar os serviços corporativos).

Tools that help in real troubleshooting

DNS Benchmark Testa performance DNS apontando o mais rápido/estável na sua rede atual.
dig Ferramenta indispensável no terminal Linux/macOS.
nslookup Busca rápida nativa. Entre usando comando limpo pra definir set type=mx.

04 Windows Commands

Network Troubleshooting
  • ipconfig /all Visualiza interface IP completa (hostname, DHCP, DNS, MAC).
  • Test-NetConnection host -Port 443 Testa portas remotas. Diagnostica firewalls bloqueando saída.
  • netstat -ano Lista conexões abertas e associa PIDs (Process IDs) locais.
Processes & Services
  • Get-Service Lista estado atual de todos os serviços via terminal.
  • Restart-Service spooler Reinicia rapidamente um serviço travado (Ex: fila de impressão).
  • services.msc Executa a janela gráfica para gerenciar todos os Serviços.
  • taskmgr / resmon Task Manager ou Resource Monitor (análise visual de disco alto).
User & AD Troubleshooting
  • Test-ComputerSecureChannel -Repair Tenta restabelecer a comunicação quebrada com o AD Domain Controller.
  • dsregcmd /status Checa status do registro da máquina perante o Entra ID (Azure AD).
Logs, Diagnostics & Disks
  • eventvwr.msc Chama o Event Viewer (Procure logs vermelhos sob System e Application).
  • diskmgmt.msc Chama Gerenciamento de Disco para resolver partições RAW ou pendentes.

Tools that help in real troubleshooting

Process Explorer (Sysinternals) Mostra processos detalhados, árvores de execução, handles e DLLs abertas.
Autoruns (Sysinternals) Analisa programas e serviços que iniciam com o Windows. (Caça malwares/lentidão).
PsExec Executa comandos remotamente em servidores corporativos na mesma rede.

05 Linux Quick Triage

Server Quick Health Checklist
  • Use when: server slow • freezing • service down • disk full • SSH slow
  • uptime Detecta tempo online (reboot inesperado?) e o Load Average original.
  • top | htop Top exibe cru. htop (se instalado) é superior pra caçar consumo de CPU.
  • free -h Análise crua e exata de consumo de memória RAM e Swap alocado.
  • df -h Exibe se o disco local/partições estourou o limite bloqueando o sistema.
  • systemctl --failed Aponta cirurgicamente quais deamons (serviços) quebraram no momento.
  • journalctl -xe Varre as últimas linhas do event log do sistema cruzando todos serviços.
Diretórios Críticos e Logs
  • /home e /tmp Perfis de usuário e lixo temporário (culpados por HD cheio).
  • /usr/bin e /var/www Binários executáveis do sistema e root web padrão (Nginx/Apache).
  • tail -f /var/log/syslog Assista os alertas gerais do servidor em tempo real rolando na tela.
  • cat /var/log/apache2/error.log Lê histórico de logs críticos caso um webserver caia.
Processes, Services, Network & SSH
  • ps aux | grep nome Isola o print de um processo alvo investigando seu PID atual.
  • systemctl list-units --type=service Lista os serviços habilitados.
  • systemctl restart nginx Reinício brusco de um daemon travado.
  • systemctl status ssh Garante que o acesso remoto não está morto.
  • netstat -tulpn Ver portas abertas escutando tráfego no kernel.
  • tail -f /var/log/auth.log Monitora tentativas de login frustradas via SSH (brute-force).
Disk Troubleshoot & Permissions
  • du -h --max-depth=1 Escaneia recursivamente do diretório atual listando as pastas mais pesadas.
  • ls -lah Detalha os bits de permissões visíveis e o dono/grupo de quem é o arquivo alvo.

Tools that help in real troubleshooting

strace Intercepta as chamadas de sistema (system calls) que um processo emperrado faz buscando arquivos invisíveis.
lsof -i / lsof -p PID Descobre portas em uso e arquivos lockados por causa raiz ou zumbis (Open Files).

06 VPN Troubleshooting

Connection & IP State
  • Connection State A interface vitual subiu no SO? Use ping IP_GATEWAY_VPN para garantir túnel estabelecido.
  • IP Assignment ipconfig /all (Win) ou ip a (Lin). A máquina pegou IP correto no Pool da empresa?
Routing & DNS
  • Routing Issues route print (Win) ou ip route (Lin). A subrede da empresa foi injetada nativamente pós-login?
  • DNS Issues nslookup server.corp. Você resolve domínios da empresa? (Conflito entre DNS Corporativo vs Wi-fi Local).
  • MTU Issues ping SERVER -f -l 1300. Sintoma: RDP/SSH cai, sistemas pesados travam (Pacote MTU grande demais engarrafando no túnel criptografado).
Auth & Certificates
  • Authentication Issues Cheque validade da senha no AD (net user xyz /domain) e fluxo de MFA (Authenticator push expirado gera Auth-Timeout).
  • Certificates / Always-On Chame certmgr.msc. O certificado pessoal/máquina foi revogado, ou CA raiz não é confiada no PC?
Testing & Signals
  • Quick Tests 1. ping 8.8.8.8 (Sua internet funciona?) → 2. Conecta VPN → 3. ping SERVER_CORP (Alcança lá?).
  • Escalation Signals Problema global caindo juntos? Provedor bloqueando UDP/IPSec (CGNAT)? Escalone com L3 (Engenharia de Redes/Firewall).

Tools that help in real troubleshooting

Client Logs (App Specifics) Foque em investigar os arquivos de LOG do cliente. Ex: Cisco AnyConnect / FortiClient / OpenVPN Logs. (Win: olhe Event Viewer).

07 Citrix Workspace Support (Endpoint)

Client-Side Troubleshooting
  • Session Launch Issues O App/Desktop clica e nada abre? Verifique cache local ou arquivo .ICA sendo associado errado no Windows.
  • Profile & Performance Lentidão extrema/Mouse travando? Valide se há picos de CPU no Notebook do usuário. (O Workspace espelha as telas usando recursos locais).
  • Network & VPN Dependency Sessão caiu "Connecting..." infinitamente? O Citrix depende 100% da VPN estar up com a port de Firewall "443" liberta pro CTX Gateway.
Critical Reset Procedures
  • Workspace → Advanced Pref. → Reset Workspace Gatilho Ouro! Resolve: Cache de ICA corrompido, perfis com configs antigas, Store quebrada e tokens inválidos locais.
  • Login Loops / Auth Failures Abre e fecha pedindo senha sem parar? Vá em Credential Manager do Win e apague senhas listadas como 'Citrix/Workspace'.
Workspace Local Client Logs
  • AppData Trace %localappdata%\Citrix\Workspace
  • ProgramFiles Trace C:\Program Files (x86)\Citrix\ICA Client
  • Flags Ocultas Gere os logs e aplique Ctrl+F buscando os erros raiz: "SSL failure", "timeout", "store unreachable", e "authentication issue".
Escalation Signals
  • Global Drops Múltiplos usuários não abrem os Apps ao mesmo tempo? Broker em pânico? Escalone à time de virtualização (L3).

Tools that help in real troubleshooting

HDX Monitor App insubstituível na máquina do analista. Varre perdas de USB redirecionado, Gráficos HDX travados, e Impressoras mortas dentro do túnel.
Citrix Director (Visão Infra) Painel Web Corporativo. Útil pelo suporte N1 para derrubar (Reset) "BrokerSessions" grudadas no Limbo do server afetando um user.

09 Virtualization Basics

Ações e Checagens Críticas
  • Snapshot Consolidation Snapshots antigos aumentam o delta disk chain → causa extremo impacto em I/O e enche silenciosamente a LUN/Datastore.
  • Hyper-V Checkpoints Acumulados aumentam o Differencing Disk (VHDX), destruindo storage local e engasgando a performance da VM hosteada.
  • VMTools / Guest Additions Instalação outdated causa problema crônico na rede, clock drift (tempo dessincronizado), mouse lag no console e recusa ao tentar "graceful shutdown".
  • Virtual Switch / VLAN ID A placa subiu mas sem internet? Verifique a vNIC para garantir que o Port Group ou a VLAN Tag está setada com ID correto.
Quick VM Troubleshooting Flow
  • Isolar problemas na orquestração: VM Down, Ping Failure, Unreachable:
  • 1. Ping VM Responde ICMP nativo ou bloqueio local de FW?
  • 2. Ping Gateway Abra o VM Console. De lá de dentro, o SO dela enxerga o roteador virtual?
  • 3. ipconfig / ip a A máquina "perdeu a placa virtual"? Ganhou um APIPA (169.254)?
  • 4. Check VMTools status Visão Infra: O pacote de tools está rodando (Running) injetando IP no painel?
  • 5. Check Snapshot usage A VM congelou porque existe rotina de backup de terceiro (Veeam) consolidando disco?
  • 6. Check Datastore space O LUN/Storage central morreu com 0 bytes e pausou todas as VMs simultaneamente?
  • 7. Check VLAN Port Group A VM migrou de pool viva (vMotion) mas a placa conectou num Switch Virtual órfão?

10 Security Quick Checks

Security Status (Win)
  • Get-MpComputerStatus Verificar saúde do Defender: AntivirusEnabled, RealTimeProtectionEnabled e SignatureLastUpdated.
  • managed-bde -status Checagem de BitLocker. Detectar se: Recovery Mode está ativo, disco criptografado parcialmente ou Protector removido.
SSH Security & Login Attempts (Linux)
  • tail -f /var/log/auth.log Obrigatório! Detecta logs de SSH rastreando: Failed password, Invalid user ou Accepted password.
  • last | lastlog Monitorar acessos fora do horário (after-hours), logins suspeitos e acessos remotos inesperados (unexpected remote IPs).
Windows Event Logs Essenciais
  • Onde buscar: Event Viewer → Windows Logs → Security:
  • Event ID 4624 Login Success (Logon efetuado perfeitamente).
  • Event ID 4625 Login Failure (Senha errada rastreável).
  • Event ID 4740 Account Locked (Conta de AD bloqueada por brute-force/typo).
  • Event ID 4672 Special Privileges (Privilégio de Administrador usado para escalar).

11 SQL Support Queries

Troubleshooting Real (Infra/Suporte)
  • Visão de Infra: Como achar o processo isolado que travou a Aplicação:
  • MSSQL: Identificar Blockings Acha a query matando o app server:
    SELECT blocking_session_id, session_id FROM sys.dm_exec_requests WHERE blocking_session_id <> 0;
  • PostgreSQL: Monitorar Atividade Acompanha queries travadas ativas:
    SELECT * FROM pg_stat_activity;
  • MSSQL: Ver tamanho das Databases Datastore/Disco apitou cheio? Veja bancos:
    EXEC sp_databases;
  • MySQL: Conexões Ativas e Presas Se o connection_pool do app estourou o limite:
    SHOW PROCESSLIST;

DBeaver Client Universal Open-Source (Multi-SGBD). Permite ao analista conectar de forma segura no Postgres/MySQL em uma janela só.
